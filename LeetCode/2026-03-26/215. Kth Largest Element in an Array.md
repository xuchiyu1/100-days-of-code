# 215. Kth Largest Element in an Array

## 中文：数组中的第 K 个最大元素
## English: Kth Largest Element in an Array

---

## 1. 题目 / Problem

给定一个整数数组 `nums` 和一个整数 `k`，返回数组中第 `k` 个最大的元素。

Given an integer array `nums` and an integer `k`, return the `k`th largest element in the array.

注意：这里不是“去重后第 k 大”，而是按数组中所有元素直接排序后的第 k 大。

---

## 2. 核心思路 / Core Idea

这题最经典的做法是：

- 用一个大小为 `k` 的最小堆
- 堆里始终保留当前最大的 `k` 个元素
- 堆顶 `heap[0]` 就是这 `k` 个元素里最小的那个
- 也就是整个数组中的第 `k` 大元素

The classic solution is:

- Use a min heap of size `k`
- Keep the current largest `k` elements in the heap
- The heap top `heap[0]` is the smallest among these `k` elements
- Therefore, it is exactly the `k`th largest element in the array

---

## 3. 为什么用最小堆 / Why Min Heap

如果用普通排序：

- 时间复杂度是 `O(n log n)`

如果用大小为 `k` 的最小堆：

- 每次只维护 `k` 个元素
- 时间复杂度更优，适合 Top K 问题

If we sort the whole array:

- Time complexity is `O(n log n)`

If we use a min heap of size `k`:

- We only maintain `k` elements at a time
- This is more efficient for Top K problems

---

## 4. 代码 / Code

```python
import heapq
from typing import List

class Solution:
    def findKthLargest(self, nums: List[int], k: int) -> int:
        heap = []

        for num in nums:
            heapq.heappush(heap, num)

            if len(heap) > k:
                heapq.heappop(heap)

        return heap[0]
```

## 6. 为什么 `heap[0]` 就是答案 / Why `heap[0]` Is the Answer

中文  
堆里始终保存的是“最大的 `k` 个数”。

例如当 `k = 2` 时：

- 堆里保存的是最大的 2 个数
- 这两个数里较小的那个，就是第 2 大
- 所以返回 `heap[0]`

English  
The heap always stores the largest `k` numbers.

For example, when `k = 2`:

- The heap stores the largest 2 numbers
- The smaller one among those two is the 2nd largest overall
- So we return `heap[0]`

## 8. 复杂度 / Complexity

设数组长度为 `n`。

- 时间复杂度 Time: `O(n log k)`
- 空间复杂度 Space: `O(k)`

Explanation:

- 每个元素最多一次 `push`
- 堆大小最多为 `k`
- 每次堆操作是 `log k`

---

## 9. 常见错误 / Common Mistakes

### 错误 1：把堆写成最大堆

这题应该用最小堆来维护前 `k` 大，不是直接维护最大堆。

This problem uses a min heap to keep the top `k` elements, not a max heap.

---

### 错误 2：忘记控制堆大小

如果不限制堆大小为 `k`，就失去了 Top K 的意义。

If you do not limit the heap size to `k`, you lose the purpose of Top K.

---

### 错误 3：返回错位置

这里应该返回 `heap[0]`，不是最后一个元素。

The answer should be `heap[0]`, not the last element.

---

## 10. 模板 / Template

```python
import heapq

def findKthLargest(nums, k):
    heap = []

    for num in nums:
        heapq.heappush(heap, num)
        if len(heap) > k:
            heapq.heappop(heap)

    return heap[0]
```
