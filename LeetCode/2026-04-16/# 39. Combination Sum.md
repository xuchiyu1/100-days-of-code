# 39. Combination Sum

**Difficulty:** Medium | **Topic:** Backtracking | **Date:** 2026-04-16

---

## 题目 / Problem

给定一个无重复元素的数组candidates和目标值target，找出所有和为target的组合，每个数可以重复使用。
Given an array of distinct integers and a target, find all combinations that sum to target. Each number may be used unlimited times.

---

## 思路 / Approach

标准回溯框架，和78子集两个关键区别：
Standard backtracking, two key differences from LC 78:

1. 记录时机：remain==0才记录，不是每层都记录
2. 递归传i不是i+1，因为同一个数可以重复选

1. Record when remain==0, not every level
2. Recurse with i not i+1, allowing reuse of same number

---

## 与46/78对比 / Comparison

    46 全排列:          78 子集:              39 组合总和:
    装满才记录          每层都记录             remain==0才记录
    传i+1              传i+1                  传i（可重复）
    用used数组          用start防重            用start防重
    无剪枝              无剪枝                 remain<0剪枝

---

## 代码 / Solution

    class Solution:
        def combinationSum(self, candidates, target):
            res = []
            track = []

            def backtrack(start, remain):
                if remain == 0:        # 恰好凑满，记录
                    res.append(track[:])
                    return
                if remain < 0:         # 超了，剪枝回头
                    return
                for i in range(start, len(candidates)):
                    track.append(candidates[i])
                    backtrack(i, remain - candidates[i])  # 传i，可重复选
                    track.pop()

            backtrack(0, target)
            return res

---

## 为什么传i不是i+1 / Why pass i not i+1

    candidates=[2,3], target=4

    传i+1（错误）:           传i（正确）:
    选2 → 下层从3开始        选2 → 下层还能选2
    漏掉[2,2]               能找到[2,2] ✅

---

## 为什么需要remain<0剪枝 / Why need remain<0

    去掉剪枝 → remain一直变负 → 无限递归 → 栈溢出
    加上剪枝 → 超过target立刻return，不继续往下走

---

## 决策树图解 / Decision Tree (candidates=[2,3,6,7], target=7)

    backtrack(0, 7)
      选2, remain=5
        选2, remain=3
          选2, remain=1
            选2, remain=-1 → 剪枝
            选3, remain=-2 → 剪枝
          选3, remain=0 → 记录[2,2,3] ✅
        选3, remain=2
          选3, remain=-1 → 剪枝
      选3, remain=4
        选3, remain=1
          选3, remain=-2 → 剪枝
      选7, remain=0 → 记录[7] ✅

## 复杂度 / Complexity

- Time: O(n^(target/min)) — 取决于target和最小元素
- Space: O(target/min) — 递归栈深度

---

## Key Takeaway

回溯三题总结：
- 46：装满记录 + used数组 + 传i+1
- 78：每层记录 + start控制 + 传i+1
- 39：remain==0记录 + start控制 + 传i（可重复）

One line: change what you record and whether to reuse = different backtracking problems.
