# 70. Climbing Stairs

**Difficulty:** Easy | **Topic:** Dynamic Programming | **Date:** 2026-04-17

---

## 题目 / Problem

爬n级台阶，每次可以走1步或2步，有多少种不同的方法？
You are climbing a staircase with n steps. Each time you can climb 1 or 2 steps. How many distinct ways can you climb to the top?

---

## 思路 / Approach

记忆化递归（自顶向下DP）
Memoized recursion (top-down DP)

到第n级 = 从第n-1级走一步 + 从第n-2级走两步
Ways to reach n = ways to reach (n-1) + ways to reach (n-2)

本质就是斐波那契数列。
This is essentially the Fibonacci sequence.

---

## 为什么需要memo / Why memo

    原始递归 fib(n) = fib(n-1) + fib(n-2):
    fib(5) 会重复计算 fib(3)两次, fib(2)三次...
    时间复杂度 O(2ⁿ)，n稍大就超时

    加了memo:
    每个n只算一次，算完存进memo，下次直接取
    时间复杂度降到 O(n)

    核心思想：用空间换时间，存住已经算过的结果

---

## 代码 / Solution

    class Solution:
        def climbStairs(self, n):
            memo = {}
            return self.helper(n, memo)

        def helper(self, n, memo):
            if n == 0 or n == 1:   # 终止条件：0级或1级只有1种走法
                return 1
            if n not in memo:      # 没算过才计算
                memo[n] = self.helper(n-1, memo) + self.helper(n-2, memo)
            return memo[n]         # 直接返回存好的结果

---

## 执行过程 / Execution (n=5)

    helper(5)
      helper(4)
        helper(3)
          helper(2) → memo[2]=2
          helper(1) → 1
        memo[3]=3
        helper(2) → memo里有！直接返回2
      memo[4]=5
      helper(3) → memo里有！直接返回3
    memo[5]=8

---

## 与斐波那契对比 / vs Fibonacci

    fib(n) = fib(n-1) + fib(n-2)   基础情况 fib(0)=0, fib(1)=1
    climb(n) = climb(n-1) + climb(n-2)  基础情况 climb(0)=1, climb(1)=1

    唯一区别：climb(0)=1（不走也算一种），fib(0)=0

---

## 复杂度 / Complexity

- Time: O(n) — 每个n只算一次
- Space: O(n) — memo存n个结果 + 递归栈

---

## Key Takeaway

DP核心：**重叠子问题 + 用memo避免重复计算**
大问题拆成小问题，小问题结果存起来，下次直接用。

DP core: overlapping subproblems + memo to avoid recomputation.
Break big problem into small ones, store results, reuse them.
