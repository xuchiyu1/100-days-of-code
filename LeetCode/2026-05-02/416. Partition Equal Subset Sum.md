# 416. Partition Equal Subset Sum

**Difficulty:** Medium | **Topic:** Dynamic Programming, 0/1 Knapsack | **Date:** 2026-05-02

---

## 题目 / Problem

给定一个只包含正整数的数组，判断是否能将数组分成两个和相等的子集。
Given a non-empty array of positive integers, determine if it can be partitioned into two subsets with equal sum.

---

## 思路 / Approach

转化问题：能不能从数组里找到一些数，加起来等于总和的一半？
Reframe: can we find a subset that sums to total/2?

这是经典0/1背包问题。
This is a classic 0/1 knapsack problem.

---

## 定义 / Definition

    dp[j] = True/False，表示能不能从数组里凑出金额j
    dp[j] = whether we can form sum j using elements from the array

---

## 状态转移方程 / State Transition

    dp[j] = dp[j] or dp[j - nums[i]]

    含义：
    - dp[j]：不用nums[i]，之前就能凑出j
    - dp[j-nums[i]]：之前能凑出j-nums[i]，加上nums[i]就是j

---

## 代码 / Solution

    class Solution:
        def canPartition(self, nums):
            sumn = sum(nums)
            if sumn % 2 != 0:      # 奇数直接False，凑不出两个相等的子集
                return False
            sumn = sumn // 2       # 目标值 = 总和的一半
            dp = [False] * (sumn + 1)
            dp[0] = True           # base case：凑出0不需要任何数
            for i in range(len(nums)):
                for j in range(sumn, -1, -1):  # 倒序！保证每个数只用一次
                    if j - nums[i] >= 0:
                        dp[j] = dp[j] or dp[j - nums[i]]
            return dp[sumn]

## dp数组变化图解 / dp Array (nums=[1,5,11,5], target=11)

    初始:  [T, F, F, F, F, F, F, F, F, F, F, F]
    用1:   [T, T, F, F, F, F, F, F, F, F, F, F]
    用5:   [T, T, F, F, F, T, T, F, F, F, F, F]
    用11:  [T, T, F, F, F, T, T, F, F, F, F, T] ← dp[11]=True ✅
    用5:   [T, T, F, F, F, T, T, F, F, F, T, T]

    return dp[11] = True

---

## 为什么倒序 / Why Reverse Order

    正序：同一个数可能被用多次（1+1+1=3，但1只有一个）
    倒序：更新dp[j]时，dp[j-nums[i]]还没被本轮更新，保证每个数只用一次

---

## 三个关键点 / Three Key Points

    1. dp[0]=True：base case，什么都不选凑出0
    2. 奇数总和直接False：无法平分
    3. 倒序遍历：每个数只用一次（0/1背包标志）

---

## 复杂度 / Complexity

- Time: O(n × sumn) — 双层循环
- Space: O(sumn) — dp数组

---

## Key Takeaway

0/1背包特征：
- dp[j]表示能不能凑出容量j
- 倒序遍历保证每个物品只用一次
- 正序遍历 = 完全背包（物品可重复用，如322零钱）

0/1 Knapsack: reverse order = each item used once
Complete Knapsack: forward order = items reusable (e.g. LC 322)
