# 322. Coin Change

**Difficulty:** Medium | **Topic:** Dynamic Programming | **Date:** 2026-04-27

---

## 题目 / Problem

给定硬币面额数组和目标金额，求凑出目标金额所需的最少硬币数。无法凑出返回-1。
Given coin denominations and a target amount, return the minimum number of coins needed. Return -1 if impossible.

---

## 思路 / Approach

记忆化递归（自顶向下DP）
Memoized recursion (top-down DP)

定义：`dp(amount)` = 凑出amount最少需要几个硬币
Definition: `dp(amount)` = min coins needed to make amount

---

## 状态转移方程 / State Transition

    dp(amount) = min(dp(amount - coin) + 1) for coin in coins

---

## 代码（无memo版，会超时）/ Without memo (TLE)

    def dp(self, coins, amount):
        if amount == 0: return 0
        if amount < 0: return -1
        res = float('inf')
        for coin in coins:
            subProblem = self.dp(coins, amount - coin)
            if subProblem == -1: continue
            res = min(res, subProblem + 1)
        return res if res != float('inf') else -1

---

## 代码（加memo，通过）/ With memo (Accepted)

    class Solution:
        def coinChange(self, coins, amount):
            memo = {}
            def dp(amount):
                if amount == 0: return 0
                if amount < 0: return -1
                if amount in memo: return memo[amount]  # 查memo
                res = float('inf')
                for coin in coins:
                    subProblem = dp(amount - coin)
                    if subProblem == -1: continue
                    res = min(res, subProblem + 1)
                memo[amount] = res if res != float('inf') else -1  # 存memo
                return memo[amount]
            return dp(amount)

---

## 为什么没有memo会超时 / Why TLE without memo

    dp(11) 会重复计算 dp(10), dp(9)...
    加了memo：每个amount只算一次，算完存起来，下次直接取
    时间复杂度从指数级 → O(amount × len(coins))

---

## 与其他DP对比 / Comparison

    70 爬楼梯:   求总数，取sum
    198 打劫:    求最大，取max
    322 零钱:    求最小，取min

    三道题都是：定义子问题 → 状态转移 → base case → memo

---

## base case / Base Cases

    amount == 0 → return 0    凑完了，不需要硬币
    amount < 0  → return -1   超了，这条路无解

---

## 复杂度 / Complexity

- Time: O(amount × len(coins)) — 每个amount只算一次
- Space: O(amount) — memo + 递归栈

---

## Key Takeaway

没有memo = 重复计算 = 超时
加了memo = 每个子问题只算一次 = 通过

DP必须memo，这是和普通递归最大的区别。
Without memo = recompute = TLE. With memo = each subproblem solved once = AC.
